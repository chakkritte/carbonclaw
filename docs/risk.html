AST & Git-Aware Refactoring Risk

Predict refactoring risk and downstream blast radius prior to modifying code using a blended heuristic of Git churn and AST parsing.

How Risk Calculation Works

CarbonClaw's risk analysis module evaluates codebases at both a structural (AST) and historical (Git metadata) level to construct a reliable index of modification hazards.

Base Risk = (Commits * 1.2) + (Authors * 2.5) + (Net Churn * 0.02)
Complexity Factor = 1 + ((Classes + Functions) / 10.0)
Risk Score = Min(100.0, Base Risk * Complexity Factor)

Heuristic Dimensions

  • Git Commits Frequency: High code churn directly increases regression probability.
  • Git Authors Density: Higher contributor counts on a single asset correlate with divergent architectural designs.
  • AST Complexity: Scales risk upward proportionally to the density of class definitions, method scopes, and function boundaries.

CLI & Chat Usage

From Terminal Command Line

Evaluate a specific source file using the CLI tool:

carbonclaw risk path/to/module.py

From Chat Sessions

Evaluate code risk interactively from the CarbonClaw console or TUI:
