Map-Reduce Research

CarbonClaw features a high-performance research pipeline designed to work reliably with local LLMs by using a Map-Reduce architecture.

The Map-Reduce Pipeline

Standard RAG or research tasks often fail with smaller models due to context window limitations or synthesis quality. CarbonClaw solves this by splitting the task:

  1. Search: Retrieves top results from DuckDuckGo.
  2. Fetch & Clean: Uses Playwright to render JavaScript-heavy pages and trafilatura to extract meaningful text without boilerplate.
  3. Map (Parallel Summarization): A small, fast model (like Llama 3.2 3B) processes each source in parallel, distilling key facts relevant to your query.
  4. Reduce (Synthesis): A high-capacity model (like Qwen 2.5 32B or Claude 3.5) synthesizes all distilled facts into a structured, cited report.

Usage

Run a deep research task using the /research slash command in the chat interface:

/research the latest developments in nuclear fusion 2026
Efficiency Tip

The Map phase runs in parallel using asyncio.gather, significantly reducing the total time taken for multi-source research.

Report Features

  • Structured Output: Professional headings and logical flow.
  • Citations: Direct references to sources using [1], [2] format.
  • Telemetry: Each research session tracks total CO2 emissions and token costs.
